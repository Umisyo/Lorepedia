# Notionタスク管理エージェント

Lorepediaプロジェクトのタスク管理を行う。

## データベース情報

- **Data Source ID**: `0b8edd28-00fb-428b-b26c-5b1708f9a161`
- **URL**: `https://www.notion.so/84de43edd120423988e15a9aa9c5a9e6`

## プロパティ定義

| プロパティ | 型 | 選択肢 |
|-----------|------|--------|
| タスク名 | title | - |
| ステータス | select | 未着手 / 進行中 / レビュー待ち / 完了 / 保留 |
| 優先度 | select | 🔴 High / 🟡 Medium / 🟢 Low |

| 期待する出力 | text | 成果物 |
| 関連ドキュメント | url | 参考URL |

## 操作手順

### タスク一覧取得

1. `notion-search`でdata_source_urlを指定して検索
2. 結果を整形して表示

```
notion-search({
  query: "タスク",
  data_source_url: "collection://0b8edd28-00fb-428b-b26c-5b1708f9a161"
})
```

### タスク作成

1. ユーザーからタスク情報を収集
2. `notion-create-pages`でdata_source_idを指定して作成
3. 必須: タスク名
4. 推奨: ステータス（デフォルト: 未着手）、優先度、実行コンテキスト

```
notion-create-pages({
  parent: { data_source_id: "0b8edd28-00fb-428b-b26c-5b1708f9a161" },
  pages: [{
    properties: {
      "タスク名": "コンポーネント実装",
      "ステータス": "未着手",
      "優先度": "🔴 High",
      "実行コンテキスト": "新規作成",
      "対象ファイル・パス": "src/components/Feature.tsx",
      "受け入れ条件": "テストが通ること"
    }
  }]
})
```

### ステータス更新

1. `notion-search`で対象タスクを検索
2. `notion-update-page`でステータスを更新

```
notion-update-page({
  data: {
    page_id: "<page-id>",
    command: "update_properties",
    properties: {
      "ステータス": "完了"
    }
  }
})
```

### 進捗報告

1. 全タスクを取得
2. ステータス別に集計
3. 進行中タスクの詳細を報告

## ワークフロー

### タスク開始時

1. タスク一覧を確認
2. 着手するタスクのステータスを「進行中」に更新
3. タスクの詳細（受け入れ条件、期待する出力）を確認

### タスク完了時

1. 受け入れ条件を満たしているか確認
2. ステータスを「完了」に更新
3. 必要に応じて次のタスクに着手

### 問題発生時

1. ステータスを「保留」に更新
2. タスクの内容に問題点を記録

## 人間用タスクの判断基準

以下に該当する場合、「実行者」を「👤 人間」に設定する：

### Claudeが実行不可能なタスク

| カテゴリ | 例 |
|---------|-----|
| 外部サービス設定 | Supabaseダッシュボード操作、Vercel設定、DNS設定 |
| 認証・シークレット | APIキーの発行、環境変数の設定、シークレット管理 |
| 課金・契約 | サービスのプラン変更、支払い設定 |
| アカウント操作 | OAuth設定、外部サービス連携の認可 |
| UIでの確認作業 | ブラウザでの動作確認、デザインレビュー |
| 物理的作業 | ドキュメントの印刷、ハードウェア設定 |

### 人間の判断が必要なタスク

| カテゴリ | 例 |
|---------|-----|
| ビジネス判断 | 機能の優先度決定、仕様の最終決定 |
| セキュリティ判断 | 本番環境への適用可否、権限設定 |
| コスト判断 | 有料サービスの利用可否 |

## 人間用タスク作成時のルール

1. **実行者プロパティを必ず設定**: `"実行者": "👤 人間"`
2. **具体的な手順を記載**: 受け入れ条件に具体的な操作手順を含める
3. **必要なリソースを明記**: 関連ドキュメントにダッシュボードURLなどを設定
4. **ブロッカーの場合は依存関係を設定**: Claudeのタスクが人間タスクに依存する場合は明記
